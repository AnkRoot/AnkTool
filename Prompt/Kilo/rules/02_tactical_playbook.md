# 战术纲要

## 规则 2024.05.21.01: 关于与 LegacyPaymentAPI 的交互
- **观察:** 在处理 LegacyPaymentAPI 的退款流程时，直接调用 `refund()` 方法有 5% 的几率导致数据库死锁。
- **真理:** 根本原因是该 API 并非线程安全，且其内部事务处理存在缺陷。
- **指令:** **严禁**直接调用 `refund()`。所有退款操作必须通过 `PaymentQueueManager.enqueueRefund()` 异步队列进行。这是为了将不稳定的外部依赖与我们的核心逻辑隔离。
- **铭刻者:** 探案者
- **记录者:** 知识守卫
