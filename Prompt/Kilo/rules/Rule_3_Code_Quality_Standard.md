# 规则3：代码质量、安全与健壮性标准

所有由 `code` 模式生成的代码都必须无条件地遵循以下标准。

## 1. Linus的简洁执念
- **缩进限制**: 如果代码需要超过3层缩进，它本身就是有问题的，必须被重构。
- **短小精悍**: 函数必须简短，只做一件事并把它做好。
- **清晰命名**: 命名应直接、无歧义，反映其真实用途。

## 2. 绝对安全与终极健壮性 (Security & Robustness by Design)
- **安全第一**: 遵循OWASP Top 10等安全实践，主动实现输入验证、依赖项扫描、避免不安全操作。
- **彻底的错误处理**: 代码**必须**包含对所有可预见异常的健壮处理，包括但不限于：空值检查、文件I/O错误、网络请求失败、超时、重试逻辑、API错误码处理等。
- **边缘情况覆盖**: 必须考虑极端输入值、并发访问、资源耗尽等边缘情况。

## 3. "好品味" (Good Taste)
- **消除特殊情况**: 优先通过重构数据结构或算法来消除 `if/else` 分支，而不是增加更多的条件判断。
- **数据结构核心**: 相信“好的程序员关心数据结构”。在编写代码前，会优先思考最合适的数据结构。
- **代码即文档**: 代码本身应该是自解释的。因此，**严禁在最终代码中添加注释**。其可读性必须通过优秀的命名、清晰的结构和简洁的逻辑来保证。
